### Recursive Function Rules: ###

When the target function is recursive, follow these rules strictly.

1. Do not expand recursive callees and do not inline recursive executions into the contract.

2. The generated ACSL must be valid Frama-C syntax. Never output QCP-only placeholders such as:
   - `Result(...)`
   - `Results(...)`

3. For a recursive C function, add a valid `decreases ...;` clause to justify termination.
   Put `decreases` inside the function contract, before `assigns` and `ensures`.

4. If the recursive function has a simple closed-form postcondition, prefer a direct contract such as:
   - `ensures \result == ...;`

5. If the recursive semantics cannot be written as a simple closed-form postcondition, introduce an explicit ACSL helper definition first, using a legal logic function or predicate, and then refer to it from `ensures`.
   The helper definition must be placed outside the function contract, before the C function declaration.

6. Any helper logic function or predicate used in the contract must be explicitly defined in ACSL. Do not use undefined logic symbols.

7. Do not output natural language, explanations, markdown fences, or invalid fallback text. Output only valid ACSL-annotated C code.

8. Do not modify the C implementation. Only repair or generate the ACSL contract.

### Recursive ACSL Guidance: ###

- Valid recursive contracts in Frama-C typically use:
  - `requires ...;`
  - `decreases ...;`
  - `assigns ...;`
  - `ensures ...;`

- If a helper logic predicate or function is needed, define it explicitly before use.
- Put helper logic definitions in a separate ACSL block before the C function, for example:
  `/*@ logic integer f(integer x, integer y) = ...; */`
- Do not place `logic`, `predicate`, or `axiomatic` definitions inside the function contract block.
- Do not use `//` comments inside ACSL blocks.

- For recursive branches, do not leave the postcondition as a placeholder. Either:
  - derive a concrete result relation, or
  - define a recursive logic predicate/function and use it legally in ACSL.

### Valid Shape Example: ###

```c
/*@ logic integer f_logic(integer m, integer n) =
      (n == 0) ? m :
      (n > 0) ? f_logic(m + 1, n - 1) :
                f_logic(m - 1, n + 1);
*/
/*@
  requires \true;
  decreases n >= 0 ? n : -n;
  assigns \nothing;
  ensures \result == f_logic(m, n);
*/
int f(int m, int n) {
  ...
}
```

### Forbidden Patterns: ###

- `ensures Results(\result);`
- `ensures Result(\result);`
- putting `logic` definitions inside a function contract block
- placing `decreases` after `assigns` or `ensures`
- any `//` comment inside ACSL
- any undefined logic function or predicate
- any prose response such as “the inputs are invalid”
