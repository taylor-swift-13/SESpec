Examples:
Use the following linked-list-oriented invariant patterns as prompt engineering guidance.

- State shape preservation with predicates when needed.
- Relate cursor variables to already-traversed and unvisited segments.
- Keep pointer validity and separation explicit.
- If the loop computes a summary, relate it to the visited prefix.

Example pattern:
```c
/*@
  loop invariant list_segment(head, p);
  loop invariant list(p);
  loop invariant \valid(p) || p == \null;
  loop assigns p, acc;
*/
```

For linked lists, prefer structural invariants plus a summary over the traversed segment.
